字符串类型是一个mutable类型（mutable类型是可变类型，在本书后面的章节会介绍这个类型#内容暂缺#），也正因为如此，字符串的拼接操作性能表现非常好；
Nim中的字符串类型是以\0结尾的（与C语言相似），Nim中的字符串类型是一个对象类型，它包含一个长度的属性（这样开发人员获取一个字符串的长度，就非常高效了，这个长度值不包含字符串结尾的\0字符），开发人员可以通过len方法获得一个字符串的长度，
另外，还有一个值得注意的地方：
if s[i] == 'a' and s[i+1] == 'b':
像这样的代码，是不用检测i+1是不是已经到了字符串的结尾的，也就是说，访问字符串的最后一个索引（\0位置的索引），不会有任何问题，也不用额外的考虑其他技术细节。
赋值运算符会导致一个字符串被复制；开发人员可以使用&运算符来链接两个字符串，也可以使用add运算符来把一个字符串添加到另一个字符串的结尾。
字符串的比较是按字符串内的字符顺序进行比较的，所有的比较操作符（比如：<，>，<=，>=等）都可以用在字符串身上。
按照惯例，所有的字符串都是UTF-8类型的字符串，但这并不是强制执行的。当你从一个二进制文件中读取数据的时候，他仅仅是一系列的字节。
s[i]是指在i位置的字符char，而不是在i位置的unichar。
string类型的变量默认值是nil，很多针对string类型的操作，都不能用在nil身上（这样做会触发一个异常）。如果你不希望这样，那么你最好用""来初始化你的字符串变量（但这会导致在堆上创建一个空字符串变量）
在Nim中，与C语言类似，反斜杠（\）是它的转义符号，比如\n代表换行符，\t代表制表符，有些时候你不想在你的字符串中大量使用转义字符，比如，你需要为一个变量设置一个路径字符串的值（就像这样的路径：C:\program files\nim）那么你可以写成这样：
var path = r"C:\program files\nim"
这就是Nim语言中的“原始字串”，在原始字串中，反斜杠不会被当作转义符。
在开发程序的时候，往往需要写大段文本，比如HTML的模版字符串，往往就是一大段文本，这时候如果写在一行内，程序就非常不容易维护和理解。Nim语言允许开发人员用成对儿的三个分号包住大段文本，文本里的反斜杠也不会被当作转义符。就像这样：
var str = """这是我的大段文本
我换了一行
我又换了一行
就算有反斜杠，\，也不会被当作转义符"""
